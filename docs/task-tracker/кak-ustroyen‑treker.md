# Как устроен трекер

Раздел **Задачи** — это специализированный раздел корпоративного портала для управления задачами в рамках проектов или управления одиночными задачами, представляющий собой таск-трекер.

**Ключевое назначение:** структурировать, распределять, контролировать и анализировать выполнение работ в команде или индивидуально.

Таск-трекер состоит из разделов:
* **Все задачи**
* **Создать задачу**
* **Статистика**
* **Мои чек-листы**
* **Проекты**
* **Настройки**
* **Шаблоны**

:::tip[Обратите внимание]
Доступ к разделам таск-трекера имеют все авторизованные пользователи корпоративного портала. &#x20;
:::

## Описание основных сущностей таск-трекера

**Задача** — это базовая единица работы, представляющая собой конкретное действие или набор действий, направленных на достижение определённого результата в рамках проекта или бизнес‑процесса.

Задачи могут быть **одиночными** и **проектными**.

**Одиночная задача** — задача не объединенная в проект.

**Проектная задача** — задача связанная с проектом.

**Шаблон** — это особый вид задачи, который не может быть взят в работу, но из которого можно создать обычную задачу.

:::tip[Обратите внимание]
&#x20;Подробнее об управлении шаблонами см. в разделе [Создание и редактирование шаблона](https://globaldrive.gitbook.io/cp/modules/tasks/sozdanie-i-redaktirovanie-shablona).&#x20;
:::

**Проект** — это целостная совокупность взаимосвязанных задач, направленных на достижение уникального результата (продукта, услуги, решения) в заданных условиях (сроки, бюджет, ресурсы, качество).

В инструментах управления работой (таск‑трекер) проект выступает как **контейнер и контекст** для задач, придавая им смысловую связность и целевую направленность.

## Жизненный цикл задачи: системные статусы задачи

Основной метрикой определяющей жизненный цикл задачи является системный статус задачи. Это обозначает, что управление любой задачей в таск-трекере осуществляется путем изменения системного статуса задачи в зависимости от стадии завершённости задачи.
Набор системных статусов задач не доступен для редактирования. Любая задача, созданная в таск-трекере, имеет набор системных статусов:

## Жизненный цикл задачи: управление проектами

В контексте управления проектными задачами необходимо понимать несколько важных моментов:

* проектные задачи имеют  те же системные статусы, что и одиночные задачи;
* проектные задачи имеют связь с проектом и дополнительные метрики, связанные со стадиями проекта;
* системные статусы задач не связаны со статусами проектов;
* стадии проекта зависят от бизнес-логики проекта;
* проект может быть использован в качестве контейнера для объединения задач из разных бизнес процессов;
* стадии проекта можно редактировать.

## Как контролировать результаты и процесс

Для отслеживания изменений в своих задачах настройте оповещения, для этого:

1. В правом верхнем углу нажмите на иконку пользователя (ваша иконка на корпоративном портале), после нажмите **Оповещения**.
2. Найдите секцию **Задачи** и проставьте необходимые опции **E-Mail** и **Портал** для каждой записи.

## Инструменты отображения

Разные форматы отображения задач (календарь, список, канбан‑доска) преследуют разные цели — вместе они дают полную картину работы c задачей в таск-трекере.

* **Календарь** — для тайм‑менеджмента и дедлайнов.
* **Список** — для быстрого обзора и массовых действий.
* **Канбан‑доска** — для визуального контроля процесса и координации команды.

Комбинация этих видов даёт **полноту картины**: можно планировать по времени (календарь), оперативно управлять списком задач и следить за ходом выполнения (канбан).

## Чек-листы

**Чек‑лист** — это структурированный список подзадач или контрольных точек внутри одной задачи (карточки) в таск‑трекере. Каждый пункт представляет собой конкретное действие или условие, которое нужно выполнить/проверить. По мере выполнения пользователь отмечает пункты (обычно галочкой), визуально отслеживая прогресс.

### Область видимости и отличия

Виды чек-листов:

* персональные чек-листы
* чек-листы, связанные с задачей

:::tip[Обратите внимание]
Подробнее об управлении чек-листами, связанными с задачами, см. в разделе [Мои чек-листы](https://globaldrive.gitbook.io/cp/modules/tasks/moi-chek-listy).&#x20;
:::

### Ключевые характеристики

1. **Вложенность**\
   Чек‑лист находится *внутри задачи либо является отдельной сущностью (персональный чек-лист)*, дополняя её описание.
2. **Декомпозиция**\
   Разбивает сложную задачу на мелкие, управляемые шаги (например, «подготовить отчёт» → «собрать данные», «написать введение», «проверить цифры»).
3. **Визуальный прогресс**\
   Отмеченные пункты выделяются (зачёркиваются, меняют цвет), что позволяет сразу увидеть, какая часть работы выполнена.
4. **Гибкость**\
   Пункты можно добавлять, удалять или редактировать в процессе работы.
5. **Необязательная последовательность**\
   В большинстве случаев порядок выполнения пунктов не жёстко фиксирован (если иное не указано явно).

:::tip[Обратите внимание]
Подробнее об управлении чек-листами, связанными с задачами, см. в разделе [Мои чек-листы](https://globaldrive.gitbook.io/cp/modules/tasks/moi-chek-listy).&#x20;
:::
